Модули

Модули позволяют структурировать код.

Модули позволяют избегать дублирования блоков кода.


Синтаксис модулей:

export/import синтаксис появился в es6


Пример экспорта по умолчанию:

// moduleOne.js const myName = () => { console.log('Eugene'); } export default myName; // moduleTwo.js import printMyName from './moduleOne.js'; printMyName();

Названия переменных могут не совпадать - это допускается при дефолтном экспорте


Модули можно запускать и без index.html, для этого нужно в нужной вам директории создать файл с расширением .mjs (example.mjs), и проверить его вызвав через node.js.
Вот как в примере таким образом: node ./src/modules/examples/example.mjs

Переименование импортов:


Node.js поддерживает es6 модули, начиная с версии 13

Также есть CommonJS модули, и их синтаксис немного отличается от ES6 модулей.
CommonJS используется для Javascript-кода в бэкенде, а ES6-модули — во фронтенде. CommonJS использует ключевое слово exports для экспорта и require для импорта, а ES6-модули используют export для экспорта и import для импорта.


Правила работы с модулями:

  1. Модули должны быть одноцелевыми
  2. Располагайте все export инструкции внизу файла
  3. Располагайте все import инструкции сверху файла
  4. По возможности используйте export default
